# Ключевые доменные сущности (Domain Entities)

В данном документе описаны основные доменные сущности системы с точки зрения Domain-Driven Design (DDD). Анализ проведен на основе ER-модели базы данных, контрактов API и бизнес-сценариев.

---

### Client (Клиент)
- **Бизнес-описание:** Основной пользователь мобильного приложения кулинарной школы. Авторизуется по номеру телефона, хранит профиль аллергий (важно для проведения классов) и управляет своими бронированиями.
- **Тип в DDD:** Aggregate Root (в контексте управления профилем и аутентификацией)
- **Ключевые атрибуты и типы данных:**

| Название | Тип данных | Обязательность | Бизнес-смысл |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | Да | Уникальный идентификатор клиента |
| `phone` | `String` | Да | Номер телефона для авторизации и связи (OTP) |
| `email` | `String` | Нет | Контактный email пользователя |
| `name` | `String` | Нет | Имя пользователя (заполняется при регистрации) |
| `allergy_profile` | `Array of Strings` | Да | Список выбранных аллергенов (тегов) для адаптации меню |

- **Связи и кратность (Relationships):**
  - `Client` 1 -> * `Booking` (У одного клиента может быть множество бронирований)
- **Матрица состояний (Жизненный цикл):**
  - *New / Unregistered:* Пользователь вводит номер, но еще не заполнил профиль (нет `name`).
  - *Active:* Профиль заполнен, может совершать действия (бронирования, оценки).

---

### ClassProgram (Программа класса)
- **Бизнес-описание:** Шаблон или учебная программа кулинарного мероприятия (например, "Итальянская паста для новичков"). На основе одной программы создаются различные слоты в расписании.
- **Тип в DDD:** Entity
- **Ключевые атрибуты и типы данных:**

| Название | Тип данных | Обязательность | Бизнес-смысл |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | Да | Уникальный идентификатор программы |
| `title` | `String` | Да | Название кулинарной программы |
| `description` | `String` | Да | Подробное описание меню и процесса |
| `complexity` | `String` | Да | Уровень сложности класса |
| `base_price` | `Integer` | Да | Базовая стоимость участия в классе на 1 человека (в копейках) |

- **Связи и кратность (Relationships):**
  - `ClassProgram` 1 -> * `Slot` (Одна программа может проводиться много раз)
- **Матрица состояний (Жизненный цикл):**
  - Управляется на стороне бэкенда (админкой). Для мобильного приложения сущность неизменяема (Read-Only) и всегда находится в актуальном статусе.

---

### Chef (Шеф-повар)
- **Бизнес-описание:** Кулинарный эксперт, который ведет класс. Его рейтинг строится на основе оценок от клиентов.
- **Тип в DDD:** Entity
- **Ключевые атрибуты и типы данных:**

| Название | Тип данных | Обязательность | Бизнес-смысл |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | Да | Уникальный идентификатор шефа |
| `name` | `String` | Да | Имя или псевдоним шефа |
| `bio` | `String` | Да | Краткая биография, регалии |
| `rating` | `Float` | Да | Средний рейтинг на основе оценок клиентов |

- **Связи и кратность (Relationships):**
  - `Chef` 1 -> * `Slot` (Один шеф проводит множество слотов расписания)
- **Матрица состояний (Жизненный цикл):**
  - Управляется на стороне бэкенда (админкой). Доступно только для чтения. Рейтинг `rating` пересчитывается сервером асинхронно после получения новых оценок в `Booking`.

---

### Slot (Слот расписания)
- **Бизнес-описание:** Конкретное проведение мастер-класса в определенное время с назначенным шефом. Хранит ресурсные ограничения: сколько доступно свободных мест и инвентаря для аренды.
- **Тип в DDD:** Aggregate Root (в контексте поиска по расписанию и контроля конкурентности)
- **Ключевые атрибуты и типы данных:**

| Название | Тип данных | Обязательность | Бизнес-смысл |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | Да | Уникальный идентификатор слота |
| `program_id` | `UUID` | Да | Ссылка на `ClassProgram` |
| `chef_id` | `UUID` | Да | Ссылка на `Chef` |
| `datetime_start` | `Timestamp` | Да | Дата и время начала класса |
| `duration` | `Integer` | Да | Продолжительность в минутах |
| `max_capacity` | `Integer` | Да | Общее/максимальное число рабочих мест |
| `available_seats` | `Integer` | Да | Текущий остаток свободных мест |
| `status` | `String` (Enum) | Да | Статус проведения |
| `address` | `String` | Да | Адрес студии |
| `available_equipment_stock` | `Integer` | Да | Доступный остаток прокатного инвентаря |
| `equipment_tariff` | `Integer` | Да | Стоимость аренды 1 единицы инвентаря (в копейках) |

- **Связи и кратность (Relationships):**
  - `Slot` * -> 1 `ClassProgram`
  - `Slot` * -> 1 `Chef`
  - `Slot` 1 -> * `Booking`
- **Матрица состояний (Жизненный цикл):**
  - `SCHEDULED`: Запланирован и доступен для просмотра.
  - `CANCELLED`: Отменен студией (форс-мажор). Недоступен для бронирования (приводит к 410 Gone при попытках записи).

---

### Booking (Бронирование)
- **Бизнес-описание:** Факт записи клиента на слот с опциональной арендой инвентаря. Главная транзакционная сущность, которая аккумулирует финансовые статусы и оценки пользователя.
- **Тип в DDD:** Aggregate Root
- **Ключевые атрибуты и типы данных:**

| Название | Тип данных | Обязательность | Бизнес-смысл |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | Да | Уникальный идентификатор брони |
| `client_id` | `UUID` | Да | Ссылка на клиента (`Client`) |
| `slot_id` | `UUID` | Да | Ссылка на слот (`Slot`) |
| `status` | `String` (Enum) | Да | Текущий статус бронирования |
| `seats_count` | `Integer` | Да | Количество мест (в MVP жестко = 1) |
| `needs_rental_equipment` | `Boolean` | Да | Включена ли аренда инвентаря в стоимость |
| `expires_at` | `Timestamp` | Нет | Дедлайн для оплаты (для статуса `PENDING_PAYMENT`, 1 час) |
| `chef_rating` | `Integer` | Нет | Выставленная оценка шефу после класса (1-5) |

- **Связи и кратность (Relationships):**
  - `Booking` * -> 1 `Client`
  - `Booking` * -> 1 `Slot`
- **Матрица состояний (Жизненный цикл):**
  - `PENDING_PAYMENT`: Создано через API, ожидается оплата перечислением. Триггер: инициация бронирования.
  - `ACTIVE`: Оплата подтверждена бэкендом. Триггер: ручная или автоматическая сверка платежа администратором.
  - `CANCELLED_BY_CLIENT`: Отменено клиентом заблаговременно или со штрафом. Триггер: пользовательское действие в UI.
  - `CANCELLED_BY_STUDIO`: Отменено студией (срыв поставки и т.д.). Триггер: решение администратора (приходит Push).
  - `COMPLETED`: Класс проведен. Триггер: наступление времени окончания слота (позволяет выставить оценку шефу `chef_rating`).
