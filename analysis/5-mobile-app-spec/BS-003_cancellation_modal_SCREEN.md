## Обзор

Подтверждающее окно (Confirmation Bottom Sheet) перед безвозвратной отменой брони.

### User Story

> Как пользователь, я хочу иметь возможность подтвердить отмену записи,
> чтобы случайно не отменить свой класс (US-160).

## Используемые запросы

### cancelBooking

**Тип:** REST  
**Метод:** POST  
**Спецификация:** `analysis/api/bookings/api.yaml` → `cancelBooking`

**Триггер:** Кнопка "Да, отменить"

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | UUID | В параметре пути (`/bookings/{id}/cancel`) |

**Обработка ответа:**
| Результат | Действие |
|-----------|----------|
| Загрузка | Спиннер на кнопке подтверждения |
| Успех (200) | Шторка закрывается. Инициатор должен обновить список бронирований. |
| Ошибка | Снекбар "Не удалось отменить запись" |

| Элемент | Описание | Действие |
|---------|----------|----------|
