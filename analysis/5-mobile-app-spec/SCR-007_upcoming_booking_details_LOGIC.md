# Логика Деталей активной брони (SCR-007)

**ID:** SCR-007_LOGIC  
**Тип:** Логика экрана  
**Домен:** 04. Бронирование  
**Приоритет:** High

---

## Обзор

Логика маппинга данных из списка бронирований и детального запроса `getSlot`, а также обеспечение бесшовного перехода из Push-уведомлений (отработка диплинков).

### User Story

> Как пользователь, я хочу получить полную информацию о своей брони по клику на Push-уведомление,
> чтобы сразу понимать, что изменилось (US-150).

---


---

## API запросы

Экран агрегирует данные. Поскольку в API нет метода `getBookingById`, при переходе по диплинку клиент запрашивает весь список активных бронирований (`getBookings`), находит нужное бронирование по ID, извлекает `slot_id` и запрашивает `getSlot` для получения адреса и названия программы.

---

## Логика состояний кнопок

- **Оплата:**
  - Кнопка "Оплатить" доступна только для статуса `PENDING_PAYMENT`.
  - Требуется проверка `expires_at`. Если время истекло — статус локально считается `CANCELLED_BY_STUDIO` (или тайм-аут оплаты), кнопка "Оплатить" скрывается.
- **Отмена:**
  - Отменить можно только классы в статусе `ACTIVE` или `PENDING_PAYMENT`.

## Связанные требования

- **FR-110** (Push-уведомление за 24 часа)
- **UC-70** (Отмена студией)
- **UC-80** (Push-напоминания)

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| 401 Unauthorized | Глобальная | Принудительный разлогин, очистка Bearer токена и перенаправление на экран авторизации SCR-001 |
| 5xx Server Error | Глобальная | Системный алерт "Сервис временно недоступен. Попробуйте позже" |
| NETWORK_ERR | Глобальная | Системный алерт "Отсутствует подключение к сети" |
