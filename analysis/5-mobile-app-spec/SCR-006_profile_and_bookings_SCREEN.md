## Обзор

Экран профиля пользователя, на котором отображаются его личные данные, выбранные аллергии, а также список предстоящих (активных) мастер-классов, на которые он записан.

### User Story

> Как пользователь, я хочу видеть свои записи и настраивать профиль,
> чтобы управлять своими планами и предпочтениями (US-130).

## Используемые запросы

### getProfile

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `analysis/api/profile/api.yaml` → `getProfile`

**Триггер:** Инициализация (PTR)

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | 200 OK | Отобразить имя, телефон и список аллергий |
| Ошибка | — | Снекбар "Не удалось загрузить профиль" |

---

### getBookings

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `analysis/api/bookings/api.yaml` → `getBookings`

**Триггер:** Инициализация (PTR)

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех | 200 OK | Список активных бронирований. Если пуст - показать Empty state "У вас пока нет записей". |
| Ошибка | — | Снекбар "Не удалось загрузить записи" |

## Элементы экрана

### 1. Блок "Мои данные"
| Элемент | Источник данных |
|---------|-----------------|
| Имя пользователя | `client.name` |
| Телефон | `client.phone` |
| Раздел "Аллергии" | Тэги из `client.allergy_profile`, кнопка "Редактировать" (открывает BS-002) |

### 2. Блок "Мои записи"
| Элемент | Источник данных / Действие |
|---------|----------------------------|
| Список карточек | Массив `Booking`. Сортировка по дате (ближайшие сверху). |
| Элементы карточки | Дата/Время слота, статус брони. (Для названия слота может потребоваться мерж с локальным кэшем или вызов `getSlot`) |
| Кнопка отмены | Если статус `ACTIVE` или `PENDING_PAYMENT` - кнопка "Отменить" (BS-003). |

## Состояния экрана

| Состояние | Условие | Отображение |
|-----------|---------|-------------|
