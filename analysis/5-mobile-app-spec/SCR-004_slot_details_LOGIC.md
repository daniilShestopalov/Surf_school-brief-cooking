# Логика Деталей Класса (SCR-004)

**ID:** SCR-004_LOGIC  
**Тип:** Логика экрана  
**Домен:** 03. Слоты и Расписание  
**Приоритет:** High

---

## Обзор

Обработка запроса на получение детальной информации о слоте, включая вложенные объекты программы и шефа. Управление состоянием кнопки CTA на основе статуса и количества мест.

### User Story

> Как пользователь, я хочу видеть полные данные слота,
> чтобы принять обоснованное решение о бронировании (US-70).

---

## Флоу

```mermaid
flowchart TD
    Start([Открытие экрана с ID]) --> FetchAPI[API: getSlot(id)]
    
    FetchAPI --> CheckResp{Успех?}
    CheckResp -->|Ошибка| ShowSnack[Снекбар: Ошибка загрузки]
    
    CheckResp -->|Успех| RenderData[Отобразить данные]
    RenderData --> CheckSeats{Места есть?}
    
    CheckSeats -->|Нет| CTADisabled[Кнопка заблокирована 'Мест нет']
    CheckSeats -->|Да| CheckStatus{Отменен?}
    
    CheckStatus -->|Да| CTACancelled[Кнопка заблокирована 'Отменен']
    CheckStatus -->|Нет| CheckBooked{Уже записан?}
    
    CheckBooked -->|Да| CTABooked[Кнопка 'Перейти к брони']
    CheckBooked -->|Нет| CTAActive[Кнопка 'Забронировать' - Открыть BS-001]
```

*(Примечание: `CheckBooked` может потребовать либо локального кэша забронированных классов из SCR-006, либо дополнительного поля в ответе API, не отраженного напрямую в `SlotDetails`, или же проверки через API бронирований)*

## API запросы

### GET /slots/{id} (`getSlot`)

**Триггер:** Инициализация экрана.

**Параметры:**
| Параметр | Тип | Источник |
|----------|-----|----------|
| `id` | UUID | Путь URL (из параметров навигации) |

**Обработка ответа:**
| Результат | Действие |
|-----------|----------|
| Успех (200) | Рендер UI, сохранение данных в стейт экрана. `SlotDetails` содержит объекты `program` и `chef`. |
| Ошибка 404 | Снекбар "Слот не найден". |
| Ошибка 500 / Сеть | Снекбар "Ошибка загрузки". |
