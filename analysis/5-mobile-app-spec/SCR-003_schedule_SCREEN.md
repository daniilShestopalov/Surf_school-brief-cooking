## Обзор

Главный экран авторизованного приложения, где отображается список доступных кулинарных классов (слотов) с возможностью фильтрации по датам.

### User Story

> Как пользователь, я хочу просматривать доступные кулинарные классы на выбранные даты,
> чтобы выбрать подходящий для себя (US-40).

## Используемые запросы

### listSlots

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `analysis/api/slots/api.yaml` → `listSlots`

**Триггер:** Инициализация (по умолчанию даты на ближайшие 7 дней), pull-to-refresh, или выбор новых дат в календаре.

**Параметры:**
| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `start_date` | date | Нет | Фильтр дат | Дата начала (по умолчанию сегодня) |
| `end_date` | date | Нет | Фильтр дат | Дата конца (по умолчанию сегодня + 7) |

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | При инициализации | Скелетон-загрузка карточек |
| Загрузка | PTR | Лоадер рефреша |
| Успех | `items` не пуст | Отобразить список карточек |
| Успех | `items` пуст | Empty state "Нет доступных классов" |
| Ошибка | 4xx / 5xx / Сеть | Error state "Не удалось загрузить расписание" + CTA "Обновить" |

## Элементы экрана

### 1. Фильтр дат

| Элемент | Описание |
|---------|----------|
| Календарная лента | Горизонтальный скролл с ближайшими датами. Активная дата выделена. |
| Кнопка календаря | Открывает модальное окно/BottomSheet для выбора диапазона дат |

### 2. Карточка слота (`items[]` из ответа)

| Элемент | Источник данных | Действие |
|---------|-----------------|----------|
| Название | `program_id.title` (в ответе это может требовать дополнения, согласно дизайну) | Открыть SCR-004 (`id`) |
| Дата и время | `datetime_start`, `duration` | — |
| Кол-во мест | `available_seats` | — |
| Статус | Если `available_seats == 0` или `status == CANCELLED` - карточка серая | — |

**Логика:**
- Карточка кликабельна всегда, но визуально отображает статус "Мест нет" или "Отменен".
