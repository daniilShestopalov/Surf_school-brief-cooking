## Обзор

Точка входа при запуске неавторизованного приложения. Позволяет авторизоваться по номеру телефона для доступа к расписанию и бронированию.

### User Story

> Как неавторизованный пользователь, я хочу авторизоваться по номеру телефона,
> чтобы получить доступ к расписанию и бронированию (US-10).

### Бизнес-ценность

- Обеспечение безопасного доступа к функционалу приложения.
- Сбор согласия на обработку ПД.

## Используемые запросы

### sendCode

**Тип:** REST  
**Метод:** POST  
**Спецификация:** `analysis/api/auth/api.yaml` → `sendCode`

**Триггер:** Нажатие на кнопку "Получить код"

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `phone` | string | Да | Ввод пользователя | Номер телефона |
| `personal_data_consent` | bool | Да | Чекбокс | Согласие на ПД |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Спиннер на кнопке отправки |
| Успех | 200 OK | Появление поля для ввода OTP, запуск таймера на повторную отправку |
| HTTP 4xx | `message` содержит текст | Снек с текстом ошибки |
| HTTP 5xx | — | Снек "Произошла ошибка. Попробуйте позже" |
| Сеть | Нет соединения | Снек "Проверьте подключение к интернету" |

---

### verifyCode

**Тип:** REST  
**Метод:** POST  
**Спецификация:** `analysis/api/auth/api.yaml` → `verifyCode`

**Триггер:** Автоматически при вводе всех цифр OTP или по кнопке

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `phone` | string | Да | Введенный ранее номер | Номер телефона |
| `code` | string | Да | Ввод пользователя | OTP код |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Спиннер проверки кода |
| Успех | `is_new_user == true` | Переход на [SCR-002](./SCR-002_registration_SCREEN.md) |
| Успех | `is_new_user == false` | Переход на [SCR-003](./SCR-003_schedule_SCREEN.md) |
| HTTP 4xx | `code == OTP_EXPIRED` | Снек "Код устарел" |
| HTTP 4xx | `code == INVALID_OTP` | Подсветка поля OTP с текстом "Неверный код" |
| HTTP 5xx | — | Снек "Произошла ошибка. Попробуйте позже" |
| Сеть | Нет соединения | Снек "Проверьте подключение к интернету" |

## Элементы экрана

### 1. Форма запроса кода

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Поле "Номер телефона" | Поле ввода телефона с маской | Ввод | Корректный номер телефона | Активация кнопки |
| Чекбокс "Согласие на обработку ПД" | Обязательный чекбокс со ссылкой | Ввод | Должен быть выбран. Ошибка: подсветка красным | Открытие документа по ссылке |
| Кнопка "Получить код" | Запрос OTP | — | Проверка телефона и чекбокса | Вызов `sendCode` |

**Логика:**
- Кнопка "Получить код" активна только при заполненном номере телефона. Если чекбокс не нажат при клике — подсветить чекбокс ошибкой.

### 2. Форма ввода OTP (появляется после `sendCode`)

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Поле ввода OTP | Поле из нескольких слотов | Ввод | 4-6 цифр (согласно дизайну) | Вызов `verifyCode` при заполнении |
| Кнопка "Запросить повторно" | Повторная отправка кода | — | Доступна по истечению таймера 60с | Вызов `sendCode` |

**Логика:**
- Поле OTP автоматически отправляет запрос при вводе последнего символа.

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** введен номер и выбран чекбокс, **Когда** нажимаю "Получить код", **Тогда** приходит СМС и появляется поле OTP | P0 |
| AC-002 | **Дано** введен OTP, **Когда** код верный и юзер новый, **Тогда** переход на SCR-002 | P0 |
| AC-003 | **Дано** введен OTP, **Когда** код верный и юзер старый, **Тогда** переход на SCR-003 | P0 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** чекбокс не выбран, **Когда** нажимаю "Получить код", **Тогда** подсвечивается чекбокс, запрос не уходит | P1 |
| AC-N02 | **Дано** введен неверный OTP, **Когда** проверяется код, **Тогда** поле подсвечивается ошибкой "Неверный код" | P0 |
