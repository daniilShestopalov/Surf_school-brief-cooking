# SCR-006: Профиль и Мои записи

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Посмотреть свои предстоящие классы, отменить запись, если нужно, и настроить аллергии (US-130).
- **С каких экранов сюда можно попасть?** Через нижнее навигационное меню (вкладка Профиль/Мои записи).
- **Куда можно уйти отсюда?** На экран деталей предстоящей брони (SCR-007) по клику на карточку, в настройки аллергий (BS-002), на шторку отмены записи (BS-003).
- **Связанные сценарии:** UC-40, UC-50.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Список карточек предстоящих (активных) бронирований. На карточках: дата, время, название, статус. При клике на карточку открываются её подробности (SCR-007). Быстрые действия на карточке (например, оплатить или отменить) (FR-30, FR-85, US-130).
- **Вторичный фокус (Medium Priority):** Блок "Мои данные": Имя, номер телефона. Раздел "Аллергии" с выбранными тегами и кнопкой редактирования (FR-25).
- **Третичный фокус (Low Priority):** Кнопка "Выйти из аккаунта". Техническая информация о версии.

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Отображаются карточки предстоящих записей и список выбранных аллергий.
- **Loading State:** Скелетон-загрузка списка записей.
- **Empty State:** Если записей нет, отображается иллюстрация "У вас пока нет записей" и CTA-кнопка "Выбрать класс" (переход на SCR-003).
- **Error State:** Снекбар при ошибке подгрузки данных профиля.
- **Validation / Invalid State:** Если статус брони "Отменена студией", карточка окрашивается в красный/серый цвет, кнопка отмены пропадает.

## 4. Ограничения и Бизнес-логика для UI
- Архив посещений (завершенные классы) в рамках MVP не выводится, только активные и предстоящие записи (FR-30).
- Запись исчезает из списка (или переходит в скрытый архив) после успешной отмены пользователем (UC-50).
