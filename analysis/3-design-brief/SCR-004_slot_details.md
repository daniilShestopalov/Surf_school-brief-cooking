# SCR-004: Детали класса

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Изучить всю информацию о классе (что будут готовить, кто шеф, где проходит) и перейти к бронированию (US-70).
- **С каких экранов сюда можно попасть?** Из расписания (SCR-003).
- **Куда можно уйти отсюда?** Вернуться в расписание, либо открыть шторку оформления бронирования (BS-001).
- **Связанные сценарии:** UC-20.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Название класса, дата и время, CTA-кнопка "Забронировать" (с базовой стоимостью на кнопке или рядом) (FR-50).
- **Вторичный фокус (Medium Priority):** Описание программы класса, уровень сложности, количество оставшихся свободных мест, имя шефа и его рейтинг (FR-50).
- **Третичный фокус (Low Priority):** Биография шефа, адрес проведения (с мини-картой), правила отмены (кратко).

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Экран заполнен данными слота. Кнопка "Забронировать" активна.
- **Loading State:** Скелетон поверх текстовых блоков, пока подгружаются расширенные данные слота (если они не кэшированы с предыдущего экрана).
- **Empty State:** Не применимо.
- **Error State:** Снекбар "Ошибка загрузки", если не удалось получить детали.
- **Validation / Invalid State:** Если мест нет (или класс отменен), CTA-кнопка "Забронировать" заблокирована (disabled) с текстом "Мест нет" / "Класс отменен".
- **Already Booked State:** Если пользователь уже записан на этот класс, CTA-кнопка меняется на "Перейти к брони" (ведёт на SCR-007) или становится "Вы уже записаны" (неактивная).

## 4. Ограничения и Бизнес-логика для UI
- Обязательно выводится индикатор свободных мест (например: "Осталось 2 места").
- CTA-кнопка привязана к нижней части экрана, чтобы всегда быть в зоне доступа большого пальца (sticky bottom bar).
