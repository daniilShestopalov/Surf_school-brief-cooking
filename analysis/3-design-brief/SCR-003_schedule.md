# SCR-003: Расписание (Главный экран)

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Ознакомиться с доступными кулинарными классами и выбрать подходящий для себя (US-40).
- **С каких экранов сюда можно попасть?** Экран по умолчанию после авторизации (SCR-001, SCR-002) или из навигационного меню.
- **Куда можно уйти отсюда?** На экран деталей класса (SCR-004) или в Профиль (SCR-006).
- **Связанные сценарии:** UC-20, UC-70.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Вертикальный или скроллящийся список карточек доступных классов (ClassProgram). На карточке: название, время, шеф, базовая стоимость, сложность, количество свободных мест (FR-50).
- **Вторичный фокус (Medium Priority):** Горизонтальный фильтр дат (ближайшие 7 дней видны сразу, кнопка календаря для выбора других дней) (FR-40, FR-45, US-50).
- **Третичный фокус (Low Priority):** Нижнее навигационное меню (Расписание, Мои записи/Профиль).

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Список карточек с актуальными слотами на выбранную дату/период.
- **Loading State:** Скелетон-загрузка карточек слотов и фильтра дат (при первом входе и смене фильтра).
- **Empty State:** Иллюстрация "Нет доступных классов" с текстом заглушкой, если на выбранные даты нет слотов (FR-40). Предложение выбрать другие даты (CTA).
- **Error State:** Плейсхолдер "Не удалось загрузить расписание" + CTA-кнопка "Обновить" при FETCH_ERROR (UC-20).
- **Validation / Invalid State:** Не применимо к этому экрану напрямую, но если класс отменен студией, карточка визуально "серая/заблокированная" с текстом "Отменен" (FR-115, UC-70).

## 4. Ограничения и Бизнес-логика для UI
- По умолчанию загружаются только классы на ближайшие 7 дней (FR-40).
- Если `available_places = 0` (или класс отменен), карточка слота должна выглядеть недоступной (disabled), но присутствовать в списке.
