# BS-001: Оформление бронирования (Bottom Sheet)

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Проверить детали перед подтверждением записи, выбрать аренду инвентаря и узнать итоговую сумму (US-80, US-90).
- **С каких экранов сюда можно попасть?** Вызывается по клику "Забронировать" с деталей класса (SCR-004).
- **Куда можно уйти отсюда?** При успешном бронировании - на экран реквизитов (SCR-005). При отмене - назад на детали.
- **Связанные сценарии:** UC-30.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Итоговая сумма к оплате, жирная CTA-кнопка "Подтвердить запись" (FR-75).
- **Вторичный фокус (Medium Priority):** Блок аренды инвентаря (чекбокс, стоимость аренды, поясняющий текст) (FR-65).
- **Третичный фокус (Low Priority):** Краткое саммари слота (дата, время, название).

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Чекбокс аренды инвентаря не выбран (или выбран). Сумма динамически пересчитывается при нажатии чекбокса.
- **Loading State:** Состояние загрузки на CTA-кнопке во время выполнения транзакции (отправки запроса на бэкенд).
- **Empty State:** Не применимо.
- **Error State:** 
  - Системная ошибка: Снекбар "Ошибка соединения, попробуйте еще раз" (NETWORK_ERR / 500 Server Error).
  - Модальное окно или снекбар: "Извините, места на этот класс только что закончились" (SLOT_FULL).
  - Снекбар: "Инвентарь уже забронирован другим. Продолжить без аренды?" (EQUIPMENT_OUT) (UC-30).
- **Validation / Invalid State:** Если `available_equipment_stock = 0`, чекбокс аренды визуально заблокирован (disabled), выводится текст: "К сожалению, весь инвентарь на это время забронирован" (FR-70, US-90).

## 4. Ограничения и Бизнес-логика для UI
- Итоговая сумма должна моментально реагировать на изменение состояния чекбокса (без задержек).
- При успехе шторка скрывается и открывается SCR-005.
