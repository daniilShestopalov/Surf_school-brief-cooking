# SCR-002: Экран регистрации (заполнение имени)

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Пользователь успешно подтвердил телефон, но является новым клиентом. Необходимо указать имя для персонального обращения шефа (US-20).
- **С каких экранов сюда можно попасть?** С экрана входа (SCR-001) после успешного ввода OTP для нового пользователя.
- **Куда можно уйти отсюда?** На главный экран расписания (SCR-003).
- **Связанные сценарии:** UC-10.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Поле ввода имени, кнопка "Продолжить" (FR-15).
- **Вторичный фокус (Medium Priority):** Приветственный текст, поясняющий зачем нам имя ("Чтобы шеф знал, как к вам обращаться").
- **Третичный фокус (Low Priority):** Отсутствует (экран максимально разгружен).

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Пустое поле ввода имени, кнопка "Продолжить" (disabled).
- **Loading State:** Спиннер на кнопке "Продолжить" при сохранении профиля.
- **Empty State:** Не применимо.
- **Error State:** Снекбар "Ошибка соединения, попробуйте еще раз" (сетевая ошибка).
- **Validation / Invalid State:** Подсветка поля ввода, если введены недопустимые символы (только буквы) или длина меньше 2 символов.

## 4. Ограничения и Бизнес-логика для UI
- Экран показывается только один раз при первичной регистрации (FR-15).
- Имя обязательно для заполнения (пропустить нельзя).
