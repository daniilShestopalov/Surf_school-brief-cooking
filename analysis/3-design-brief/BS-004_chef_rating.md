# BS-004: Оценка шефа (Bottom Sheet / Card)

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Оставить обратную связь после посещения класса (US-140).
- **С каких экранов сюда можно попасть?** Открывается автоматически поверх приложения по Push-уведомлению, либо висит как карточка на главном экране (SCR-003) / в профиле (SCR-006) в течение 24 часов после занятия.
- **Куда можно уйти отсюда?** Скрыть шторку после оценки или закрыть по крестику.
- **Связанные сценарии:** UC-60.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** 5 звезд для оценки (от 1 до 5) (FR-100).
- **Вторичный фокус (Medium Priority):** Вопрос "Как вам класс <Название> и шеф <Имя>?", фото шефа.
- **Третичный фокус (Low Priority):** Кнопка "Оценить позже" или крестик для закрытия (FR-105).

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Звезды не закрашены. При тапе на звезду, закрашиваются до выбранной и сразу (или по кнопке "Отправить") уходит запрос.
- **Loading State:** Спиннер внутри карточки при отправке оценки.
- **Empty State:** Не применимо.
- **Error State:** Снекбар "Время для выставления оценки вышло" (RATING_TIMEOUT) и шторка закрывается (UC-60).
- **Validation / Invalid State:** Отсутствие возможности написать текстовый отзыв (строго по FR-100).

## 4. Ограничения и Бизнес-логика для UI
- Нет текстового поля (FR-100).
- Элемент исчезает навсегда после успешной отправки оценки или через 24 часа после класса (FR-105, UC-60).
