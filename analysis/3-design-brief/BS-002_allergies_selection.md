# BS-002: Выбор аллергий (Bottom Sheet)

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Указать свои аллергии, чтобы шеф учел это при подготовке (US-60).
- **С каких экранов сюда можно попасть?** По клику на блок аллергий в Профиле (SCR-006).
- **Куда можно уйти отсюда?** Возврат в Профиль.
- **Связанные сценарии:** UC-40.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Облако или список тегов аллергенов (Глютен, Лактоза, Орехи и т.д.), кнопка "Сохранить" (FR-25).
- **Вторичный фокус (Medium Priority):** Краткий пояснительный текст ("Выберите продукты, на которые у вас аллергия").
- **Третичный фокус (Low Priority):** -

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Теги, которые уже были сохранены, визуально выделены (selected).
- **Loading State:** Спиннер при загрузке справочника тегов с сервера, лоадер на кнопке при сохранении.
- **Empty State:** Если справочник на бэкенде пуст, отобразить заглушку "Список аллергенов пока пуст".
- **Error State:** Снекбар "Не удалось сохранить изменения. Попробуйте позже" (SAVE_ERROR) или экран "Не удалось загрузить справочник", если сервер вернул ошибку 500/сеть при первичной загрузке.
- **Validation / Invalid State:** Не применимо.

## 4. Ограничения и Бизнес-логика для UI
- Множественный выбор (Multi-select) тегов.
- Изменения применяются только после нажатия "Сохранить", а не по клику на тег.
