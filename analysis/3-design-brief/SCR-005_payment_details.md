# SCR-005: Инструкция по оплате

## 1. Контекст и Цель пользователя
- **Зачем пользователь приходит на этот экран?** Получить банковские реквизиты для перевода и увидеть, сколько у него есть времени до сгорания брони (US-100).
- **С каких экранов сюда можно попасть?** После успешного создания брони (из BS-001) или при нажатии кнопки оплаты из экрана деталей активной брони (SCR-007).
- **Куда можно уйти отсюда?** На главный экран (SCR-003), в Профиль (SCR-006) или назад к деталям брони (SCR-007).
- **Связанные сценарии:** UC-30.

## 2. Информационная Архитектура (Что должно быть на экране)
- **Первичный фокус (High Priority):** Итоговая сумма перевода, номер карты/СБП (с кнопкой "Скопировать"), обратный таймер на 1 час (FR-80, BR-40).
- **Вторичный фокус (Medium Priority):** Инструкция ("Переведите точную сумму, укажите в комментарии номер брони..."), статус брони "Ожидает оплаты".
- **Третичный фокус (Low Priority):** Кнопка "Перейти в мои записи".

## 3. Матрица Состояний Интерфейса (UX States)
- **Default State:** Таймер идет, реквизиты отображаются.
- **Loading State:** Не применимо (данные уже получены).
- **Empty State:** Не применимо.
- **Error State:** Не применимо.
- **Validation / Invalid State:** При истечении таймера экран визуально меняется: таймер 00:00, текст красным "Время вышло, бронь аннулирована" (если пользователь не покинул экран).

## 4. Ограничения и Бизнес-логика для UI
- Таймер должен отображаться в формате MM:SS и синхронизироваться со временем создания брони (бэкенд).
- Кнопка "Скопировать" должна явно давать визуальный отклик (Toast: "Реквизиты скопированы").
